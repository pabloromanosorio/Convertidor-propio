**Create `src/editor/web_verifier.js`:**

**Function: `askQuestion(question, findingContext, modelKey, log)`**

Uses OpenRouter with web search plugin to answer translation-related questions.

1. Build system prompt:
```
You are a translation and linguistics expert assistant with web search capabilities. You help translators verify terminology, check authoritative sources, and resolve translation questions.

CONTEXT:
Source language: {srcLang}
Target language: {tgtLang}
Source text: "{findingContext.sourcePhrase}"
Current translation: "{findingContext.contextPhrase}"
Error identified: "{findingContext.errorPhrase}" ({findingContext.errorType})
Proposed correction: "{findingContext.replaceText}"

When answering:
1. Search for authoritative sources (official terminology databases, style guides, RAE, Fundéu, domain-specific glossaries)
2. Cite your sources with URLs when possible
3. If the question is about terminology, check if there's an official or standard translation
4. If the question is about grammar, reference the specific rule from RAE or similar authority
5. Be concise and directly useful for the translator's decision
```

2. Build user message: the user's question

3. Call OpenRouter with web search plugin enabled:
```javascript
const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: resolvedModelId,  // Use the OpenRouter model ID
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: question }
    ],
    plugins: [{ id: 'web' }],  // Enable web search
    temperature: 0.3
  })
});
```

4. Parse response and return `{ answer, sources: [], usage }`

**Function: `verifyFinding(finding, store, modelKey, log)`**

Automated verification — asks a pre-built question about the finding:
```
Is the following correction accurate for {tgtLang} translation?
Original: "{finding.findText}"
Proposed: "{finding.replaceText}"
Error type: {finding.errorType}
Please verify against authoritative linguistic sources.
```

Returns `{ verified: boolean, confidence: string, note: string, sources: [] }`

**Add to `server.js`:**

**`POST /api/editor/session/:sessionId/ask`**
- Body: `{ findingId, question, modelKey }`
- Gets finding context from store
- Calls `askQuestion(question, findingContext, modelKey, broadcastLog)`
- Returns `{ answer, sources }`

**`POST /api/editor/session/:sessionId/finding/:findingId/verify`**
- Calls `verifyFinding(finding, store, modelKey, broadcastLog)`
- Updates finding with `verificationNote`
- Returns `{ verified, confidence, note, sources }`

**Web search UI per finding card:**

Add a small "🔍 Verify" button to each finding card's action bar (next to Approve/Reject).

Clicking "🔍 Verify" expands a panel below the finding card:

```
┌─────────────────────────────────────────────────────┐
│ 🔍 Ask about this finding                          │
│                                                     │
│ ┌─────────────────────────────────────────────────┐ │
│ │ [Question input: e.g., "Is 'desechará' the      │ │
│ │  correct term in Colombian legal Spanish?"]      │ │
│ └─────────────────────────────────────────────────┘ │
│ [Ask]  [Auto-Verify]                                │
```

- Verification badge appears on the finding card: ✅ Verified / ❌ Not verified / 🔍 (pending)

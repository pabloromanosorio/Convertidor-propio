**Diff visualization function: `renderDiff(findText, replaceText, contextPhrase)`**

This shows the FULL context phrase with the find/replace changes highlighted:

1. Find the `findText` within `contextPhrase`
2. Split context into: before + findText + after
3. Compute a word-level diff between `findText` and `replaceText`:
   - Split both into words
   - Use a simple LCS (Longest Common Subsequence) algorithm to identify: words only in find (removals), words only in replace (additions), common words
   - Generate HTML: common words in normal text, removed words in `<span class="diff-removed">` (pastel red background + strikethrough), added words in `<span class="diff-added">` (pastel green background)
4. Render: `before` + diff visualization + `after`
5. Below the context diff, show raw Find/Replace as monospace text

**Implement simple word-level LCS diff:**
```javascript

**Consistency check UI:**

When user clicks "Run Consistency Check" button:
1. POST `/api/editor/session/{sessionId}/consistency-check` with `{ modelKey }`
2. Show loading indicator
3. On response, if contradictions found:
   - Show a modal or inline warning panel ABOVE the findings
   - For each contradiction:
     ```
     вҡ пёҸ Inconsistency Detected
     Findings {id1} and {id2} propose different corrections for the same pattern.
     Reason: {reason}
     Recommendation: {recommendation}
     [Accept Recommendation] [Dismiss]
     ```
   - "Accept Recommendation" updates the affected findings' replaceText to the recommended value
   - "Dismiss" leaves them as-is (translator decides manually)
4. If no contradictions: Show success message "вң… All corrections are consistent"

Finding cards with consistency warnings show an orange border and a warning icon with tooltip showing the reason.

**Apply & Export workflow:**

"Apply & Export" button (enabled only when at least 1 finding is approved):
1. Show confirmation dialog:
   ```
   Apply {N} approved corrections?
   {M} findings were rejected and will be skipped.
   {K} findings are still pending review.
   
   [Cancel] [Apply & Export]
   ```
2. If pending findings exist, warn: "вҡ пёҸ {K} findings have not been reviewed yet. Continue anyway?"
3. On confirm:
   a. POST `/api/editor/session/{sessionId}/apply` вҖ” applies corrections to XLIFF
   b. Show result: "{N} corrections applied, {F} failed"
   c. If any failed, show details: "Could not apply: '{findText}' in segment {id} вҖ” text changed by prior correction"
   d. POST `/api/editor/session/{sessionId}/export` вҖ” exports corrected DOCX
   e. Show download link: "рҹ“Ҙ Download Editado - filename.docx"
4. After export, show summary:
   ```
   вң… Export Complete

   - Failed: 2 (see details)
   - Download: [Editado - Contract_ES.docx]
   - [Start New Review] [Back to Home]

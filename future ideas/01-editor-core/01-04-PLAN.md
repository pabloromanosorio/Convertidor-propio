**Create `public/editor.js`:**

**File upload handling:**
- `handleSourceDrop(event)` / `handleTargetDrop(event)` — store File objects, show filename in drop zone, enable Start button when both present
- `handleXliffDrop(event)` — store XLIFF file, disable DOCX drop zones, enable Start button
- `startAnalysis()`:
  1. Build FormData with files + model + customPrompt
  2. POST to `/api/editor/upload` — receives `{ sessionId, sections, totalSegments, metadata }`
  3. Store sessionId
  4. Show review area, hide upload area
  5. Populate section navigator from `sections` array
  6. Auto-start analysis: POST to `/api/editor/analyze` with `{ sessionId, modelKey, customPrompt }`
  7. Connect SSE for progress streaming

**SSE integration:**
- Reuse same EventSource pattern from script.js
- Connect to `/api/logs` (same endpoint)
- Display progress in the progress bar area
- Update section status indicators as each section completes

**Section navigator:**
- `populateSections(sections)` — creates section list items in sidebar
- `selectSection(sectionIndex)`:
  1. Highlight in sidebar
  2. GET `/api/editor/session/{sessionId}/section/{sectionIndex}` — load segments
  3. Populate segment browser with source-target pairs
  4. GET `/api/editor/session/{sessionId}/section/{sectionIndex}/findings` — load findings (if analysis complete)
  5. Populate findings panel (delegate to Plan 05's render function)
- Auto-select first section after upload

**Segment browser:**
- `renderSegments(segments)` — creates two-column source-target display
- Each segment row is clickable — stores segmentId for cross-referencing with findings
- Segment IDs displayed as small badges

**Model dropdown:**
- Create new ModelDropdown instance (same class from script.js)
- Load models from `/api/models` on page load
- Store selected model key

**Preset management:**
- `loadEditorPresets()` — GET `/api/editor/presets`
- `saveEditorPreset(name, prompt)` — POST `/api/editor/presets`
- `deleteEditorPreset(name)` — DELETE `/api/editor/presets/:name`
- Populate preset dropdown, wire save/delete buttons

**Add to `server.js`:**

Add editor preset routes (same pattern as translation presets):
```
const editorPresetsPath = path.join(userPresetsDir, 'editor_presets.json');

GET /api/editor/presets — load editor presets
POST /api/editor/presets — save editor preset

```

**i18n:** Add editor-specific translation strings to a small `editorTranslations` object in editor.js (same pattern: `en` and `es` keys). Use `data-i18n` attributes on HTML elements. At minimum translate: section titles, button labels, status messages.
